# PRD: 트랜스크립트 뷰어 (통합 타임라인·툴 상세·모바일)

## 달성 가치
- V1: 대화 구조의 시각적 이해 — 메인/서브에이전트 통합 타임라인, 툴 호출 상세, 모바일 가독성

## Acceptance Criteria

### VW-AC1: 통합 타임라인
- **설명**: 메인 세션과 서브에이전트 메시지를 하나의 타임라인에 시간순으로 렌더링한다.
  서브에이전트 메시지는 호출 지점(invocation point)에 인라인으로 삽입된다.
  서브에이전트가 없는 세션도 정상 렌더링한다.
- **달성 가치**: V1
- **검증 방법**: 서브에이전트 포함/미포함 픽스처로 타임라인 순서와 인라인 삽입 위치 확인

### VW-AC2: 메인/서브에이전트 시각적 구분
- **설명**: 메인 메시지와 서브에이전트 메시지를 시각적으로 구분하고,
  서브에이전트 메타데이터를 콘텐츠와 함께 인라인 표시한다.
- **달성 가치**: V1
- **검증 방법**: 두 메시지 유형의 구분 스타일과 메타데이터 표시를 UI에서 확인

### VW-AC3: 확장/축소 인터랙션
- **설명**: 서브에이전트·툴 상세를 클릭으로 확장/축소할 수 있다.
  확장/축소 시 타임라인 위치와 스크롤 위치가 유지되고, 키보드로 타임라인 항목을 탐색할 수 있다.
- **달성 가치**: V1
- **검증 방법**: 확장/축소 전후 스크롤·위치 유지 및 키보드 내비게이션 동작 확인

### VW-AC4: 툴 호출 표시
- **설명**: `tool_use` 메시지는 타임라인에 툴 이름을 인라인으로 표시한다.
  Task 툴은 `Task [subagent_type]` 형식(예: `Task [code]`), subagent_type이 없으면 `Task`만,
  비-Task 툴은 이름만 표시한다. 클릭 시 상세 뷰에서 툴 입력을 포맷된 JSON으로 표시하고
  다시 클릭하면 축소된다.
- **달성 가치**: V1
- **검증 방법**: Task/비-Task 툴 각각의 인라인·상세 뷰 표기와 JSON 포맷 확인

### VW-AC5: 긴 텍스트 절단과 타임스탬프
- **설명**: 긴 텍스트는 절단(truncation)하여 타임라인 가독성을 유지하고,
  각 메시지에 타임스탬프를 표시한다.
- **달성 가치**: V1
- **검증 방법**: 장문 픽스처에서 절단 동작, 메시지별 타임스탬프 렌더링 확인

### VW-AC6: 모바일 레이아웃
- **설명**: 모바일 뷰포트(375×667, 360×640)에서 툴 상세/입출력 박스에 컴팩트 패딩과
  11px 코드 폰트, `pre-wrap`을 적용해 가로 스크롤이 발생하지 않는다.
  641px 이상 데스크톱에서는 기존 패딩과 14px 폰트를 유지하며, 뷰포트 리사이즈에도 일관된다.
- **달성 가치**: V1
- **검증 방법**: 각 뷰포트에서 패딩·폰트·overflow 계측, 리사이즈 전환 확인
