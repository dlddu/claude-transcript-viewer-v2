# Claude Transcript Viewer v2 제품 가치 문서

> 이 문서는 모든 하위 문서(PRD, AC, 테스트)의 최우선 판단 기준이다.

## 제품 소유자
- **(미정)** — 소유자 확정 시 이 항목을 갱신할 것.
  소유자가 없는 동안 모든 가치는 고아 가치(orphan value) 상태이다. (상태 추적 문서 참조)

## 제품 가치

### V1: 대화 구조의 시각적 이해
- **유형**: 추상적
- **설명**: Claude 대화 기록(JSONL 트랜스크립트)을 서브에이전트 호출까지 포함해
  하나의 타임라인으로 시각화하여, 사용자가 세션의 흐름·도구 호출·서브에이전트
  동작을 구조적으로 이해할 수 있게 한다. 데스크톱과 모바일 모두에서 읽기 좋아야 한다.

### V2: 즉각적인 대화 탐색
- **유형**: 구체적
- **설명**: 세션 ID 또는 메시지 UUID만으로 원하는 트랜스크립트를 즉시 찾아 연다.
  붙여넣은 메시지 원문에서 UUID를 자동 추출하는 수준의 낮은 입력 마찰을 목표로 하며,
  실패 시(미등록 세션, UUID 없음) 명확한 피드백을 제공한다.

### V3: 트랜스크립트 크기에 무관한 가벼움
- **유형**: 구체적
- **설명**: 트랜스크립트 바이트가 백엔드를 통과하지 않는다. 브라우저가 단기 presigned URL로
  S3에서 직접 다운로드하고 로컬에서 파싱·렌더링하므로, 트랜스크립트 크기와 무관하게
  백엔드 메모리·대역폭이 일정하다.

### V4: 운영 부담 최소화
- **유형**: 추상적
- **설명**: 개인/소규모 운영을 전제로 인프라를 최소화한다. 단일 Docker 이미지,
  단일 Kubernetes 워크로드, 외부 DB 없이 SQLite 인덱스만 사용하며,
  삭제 등 파괴적 작업은 중단되어도 안전하게 재시도 가능해야 한다.
