# PRD: 세션 룩업 (Session ID · Message UUID)

## 달성 가치
- V2: 즉각적인 대화 탐색 — 두 가지 진입 경로(세션 ID 직접 입력, 메시지 원문에서 UUID 자동 추출)와 명확한 실패 피드백

## Acceptance Criteria

### LK-AC1: 룩업 탭 UI
- **설명**: 메인 페이지에 "Message UUID"·"Session ID"·"Sessions" 세 탭을 표시한다.
  이 중 룩업(식별자로 열기) 경로는 "Message UUID"·"Session ID" 두 탭이며,
  세 번째 "Sessions" 탭(브라우징 목록)은 session-list PRD의 SL-AC2 소관이다.
  기본 활성 탭은 "Message UUID"이며, 탭 전환 시 해당 입력 UI가 표시되고
  각 룩업 탭에서 룩업 플로우를 끝까지 완료할 수 있다.
- **달성 가치**: V2
- **검증 방법**: 탭 렌더링(세 탭)·기본 활성 상태(Message UUID)·전환·룩업 플로우 완료를 UI에서 확인

### LK-AC2: Session ID 룩업
- **설명**: 세션 ID 입력 필드와 룩업 버튼을 제공한다. 유효한 세션 ID 입력 시 트랜스크립트를
  로드하고 메타데이터·서브에이전트 섹션을 표시한다. 로딩 상태 표시, 입력 포맷 사전 검증,
  공백 trim, Enter 키 제출, 최초 조회 후 다른 세션 재검색을 지원한다.
- **달성 가치**: V2
- **검증 방법**: 유효/무효 입력, 로딩 상태, Enter 제출, trim, 재검색 각 케이스 확인

### LK-AC3: Message UUID 추출 룩업
- **설명**: textarea에 붙여넣은 메시지 원문에서 UUID v4를 자동 추출해(추출 성공 시 그린 배지 표시)
  해당 트랜스크립트를 로드한다. 빈 입력 시 "Extract & Search" 버튼은 비활성화되고,
  placeholder 힌트를 표시하며, Ctrl+Enter 단축키로 추출·검색을 실행할 수 있다.
- **달성 가치**: V2
- **검증 방법**: UUID 포함 원문 붙여넣기 → 배지 → 로드 플로우, 버튼 비활성, 단축키 확인

### LK-AC4: 실패 피드백
- **설명**: 세션 ID 미존재 시 API 에러 메시지를, 입력에 UUID v4가 없으면 "No UUID found" 에러를,
  저장된 세션이 없을 때는 안내 메시지를 표시한다.
- **달성 가치**: V2
- **검증 방법**: 미존재 세션·UUID 없는 입력·빈 상태 각각의 에러/안내 메시지 확인
